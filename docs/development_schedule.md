# PPlan 개발 계획서

## 1. 개발 목표

개발 기획 문서에 명시된 주요 기능을 안정적으로 구현하고, 사용자에게 쾌적한 사용 경험을 제공하는 고품질의 모바일 애플리케이션을 2.5개월 내에 출시하는 것을 목표로 합니다.

## 2. 개발 단계 및 일정 (총 10주)

| 단계 | 주차 | 주요 작업 내용 | 담당자 (예시) |
| :--- | :--- | :--- | :--- |
| **1. 환경 설정 및 기본 구조** | 1주차 | - Flutter 프로젝트 생성 및 기본 설정<br>- Firebase 연동 및 초기 설정<br>- Isar 로컬 DB 스키마 설계 | 공통 |
| **2. 핵심 기능 구현 (1)** | 2-3주차 | - 타임라인 데이터 모델 및 UI 구현<br>- 사진 라이브러리 연동 및 이미지 표시 | Frontend |
| **3. 핵심 기능 구현 (2)** | 4-5주차 | - 스크랩북 생성 및 편집 기능 구현<br>- 여행 관리 기능 모델 및 UI 구현 | Frontend |
| **4. 백엔드 연동** | 6주차 | - Firebase Authentication을 이용한 회원가입/로그인<br>- Firestore를 이용한 데이터 동기화 로직 구현 | Backend/Fullstack |
| **5. 고도화 및 테스트** | 7-8주차 | - UI/UX 개선 및 애니메이션 추가<br>- 단위/통합 테스트 코드 작성 및 QA 진행 | Frontend/QA |
| **6. 안정화 및 배포 준비** | 9-10주차 | - 버그 수정 및 성능 최적화<br>- 앱 스토어 등록 정보 준비 및 스크린샷 촬영 | 공통 |

## 3. 마일스톤

*   **1개월 차:** 핵심 기능(타임라인, 스크랩북) 프로토타입 완성
*   **2개월 차:** Firebase 연동 및 데이터 동기화 기능 완성
*   **2.5개월 차:** 앱 스토어 출시

## 4. 협업 및 커뮤니케이션

*   **버전 관리:** Git-flow 전략을 사용하여 브랜치를 관리합니다.
*   **이슈 관리:** GitHub Issues를 사용하여 작업을 할당하고 진행 상황을 추적합니다.
*   **정기 회의:** 주 1회 정기 회의를 통해 진행 상황 공유 및 이슈를 논의합니다.
